Sass for Grunt

It was finally time to learn a CSS preprocessor. I had heard of SASS and LESS, and one of the top responses to a quick google search of "sass vs less" was a css-tricks article that favored SASS (but only slightly).

Installation, aka: Why do I have to install Ruby?

Its because the preprocessing is done by a ruby program. Its good that its pretty easy to install. Go grab an installer then open up a terminal and type out gem install sass (yes even in windows). You might get a strange error that says something about an SSL certificate. If so type gem source -a "http://rubygems.org" into the terminal. Its not recomended but its the only workaround besides reverting to an earlier version of ruby gems.

Grunt configurations

Its pretty easy to do this too. I love being plesantly surprised. This is where you'll find most of hte instructions. But I will outline the important stuff.

  1. In a terminal in your project folder type npm install grunt-contrib-sass --save-dev.
  2. In your grunt file if you don't already have require('load-grunt-tasks')(grunt); (which adds all of the grunt-contrib-xxx modules), you will need to add grunt.loadNpmTasks('grunt-contrib-sass');
  3. Make a new grunt task called sass like below.
    sass: {
      dist: {
        files: {
          'path/to/main.css': 'path/to/main.scss'
        }
      }
    }
    This is the simplest form of the grunt-contrib-sass preprocessor. It simply compiles everything in main.scss into the css file main.css. Dont worry if you see main.css.map get created too, its just all part of the processing.
  4. In your serve grunt task add 'sass', and run grunt serve in a terminal.
